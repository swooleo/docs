    Coroutine::defer

    Coroutine::defer用来在协程关闭之前(即协程函数执行完毕时)调用被defer注册的代码。

    注意，因为defer一般用于资源的释放，所以它的调用顺序是先注册defer的后执行。逆序符合资源释放的正确逻辑, 后申请的资源可能是基于先申请的资源的, 如先释放先申请的资源, 后申请的资源可能就难以释放。另外需要注意的是，在协程内抛出了异常，已注册的defer也会被执行。

    方法原型

    Swoole\Coroutine::defer

<?php
public static function defer($callback) { }
@param callable $func

@return mixed

    这个方法有一个别名，defer：

    defer

<?php
function defer($callback) { }
@param callable $func

@return mixed

    参数

    callable $func

    代表一个PHP函数或者方法。

    普通函数

    <?php

function task1()
{
    echo 'task1' . PHP_EOL;
}

function task2()
{
    echo 'task2' . PHP_EOL;
}

Swoole\Coroutine\run(function () {
    go(function () {
        echo '1' . PHP_EOL;
        Swoole\Coroutine::defer('task1');
        Swoole\Coroutine::defer('task2');
        echo '2' . PHP_EOL;
    });
});

    使用协程的返回值作为参数

    <?php

Swoole\Coroutine\run(function () {
    go(function () {
        echo '1' . PHP_EOL;
        Swoole\Coroutine::defer(function ($param) {
            var_dump($param);
        });
        echo '2' . PHP_EOL;
        return "swoole";
    });
});
